# changelog
changelog=Version 1.5.1:
  - Centralised distance/KP measurement via a single helper so all tools consistently use the project ellipsoid (with a WGS84 fallback when unset). Every QgsDistanceArea in the plugin now flows through this helper.
  - KP-emitting processing algorithms gained an Ellipsoidal / Cartesian distance-mode option (default Ellipsoidal; no behaviour change).
  - KP Plotter now measures KP in the line-layer CRS and reprojects the on-map marker; reference and route layers no longer need matching CRSes.
  - Find Nearest KP and Place KP Points (raster sampling) now auto-reproject mismatched inputs with a feedback warning instead of refusing to run.
  - Translate KP Between RPLs: clearer same-CRS error message that names both layers' CRSes.
  - KP Mouse Tool: range ring is now drawn as a true geodesic circle so its edge sits on the cursor regardless of the project CRS scale factor (fixes drift in Web Mercator etc.). Cartesian-mode distance area is now well-defined on QGIS >= 3.30. Three remaining .exec_() dialog calls replaced with .exec() for Qt6.
  - Fixed: RPL Route Comparison was missed by the centralisation pass and silently fell back to planar distances when the project ellipsoid was unset.
  - Fixed: bare except: clauses in Calculate Seabed Length narrowed; Dynamic Buffer (Lay Corridor) group label corrected ("Other tools" -> "Other Tools") so it no longer creates a duplicate toolbox group.
  - Reference-line input labels standardised to "Reference Line Layer" across processing algorithms (parameter IDs unchanged).
  - Catenary Calculator (v1) marked as Legacy; planned for removal in 1.6.
  - Renamed "Import Bathy MDB" to "Import MDB" (it already imports any GeoMedia feature class, not just bathymetry); algorithm id unchanged so saved models still work.
  - QGIS 4.0 / Qt6 readiness: matplotlib backend, sip and resources.py imports use the qgis.PyQt shim; deprecated FlagAdvanced replaced; metadata declares supportsQt6=True.
  - Removed: unused subsea_cable_tools_dialog.py / .ui and the dead run() method on the main plugin class.
  - Internal: processing provider now grouped + sorted alphabetically; lib/ only added to sys.path when the host environment is missing a vendored library.

Version 1.5.0 (2026-04-26): Many new processing tools and tool improvements (Extract KP Ranges Rule Based, Identify Features Intersecting RPL, RPL Route Comparison, Translate KP Between RPLs, Identify RPL Crossing Points, Identify RPL Area Listing, Merge/Group KP Range tables, KP Range Depth + Slope Summary, Identify Hazards in Lay Corridor, Export Chartlets, Plot Line Segments from Table, Extract A/C Points, Extract Lines Intersecting Polygons), Catenary Calculator V2, ellipsoidal/cartesian KP Mouse Tool, Depth Profile improvements, more flexible MDB import. Some in-development tools from unreleased internal builds have been withdrawn pending further work.
